1 Â· What is Jennah CLI?

jennah is a command-line tool for managing workload deployment jobs on the Jennah platform. You can create tenants, submit jobs, check their status, update them, and delete them â€” all from your terminal.

Think of a tenant as a user or team account, and a job as a task or workload that belongs to that tenant.

4 Â· Tenant Commands

Tenants must be created before jobs can be submitted. Each tenant has a name, an email, and an auto-generated UUID as its ID.

Create a tenant
Registers a new tenant. The Tenant ID is generated automatically.
.\jennah tenant create --name <name> --email <email>

Delete a tenant
Permanently deletes a tenant and all of its jobs. This cannot be undone.
.\jennah tenant delete --tenant-id <tenant-id>

Example

.\jennah tenant delete --tenant-id 4a3f92bc-1234-...

5 Â· Job Commands

All job commands require --tenant-id. Get your tenant ID by running jennah tenant list.

Submit a job
Creates a new job under the given tenant. Returns a Job ID.
.\jennah submit --tenant-id <id> [--status <status>]

Flags

FlagRequiredDefaultDescription
--tenant-idYesâ€”The tenant that owns the job
--statusNoPENDINGInitial status of the job

Examples

.\jennah submit --tenant-id 4a3f92bc-...
.\jennah submit --tenant-id 4a3f92bc-... --status RUNNING

Output

âœ“ Job submitted successfully
  Job ID:  16f2379d-746b-4e50-986b-f8cb7f1d3ab6
  Status:  PENDING
  Tenant:  4a3f92bc-...
List jobs
Lists all jobs for a tenant. Optionally filter by status.
.\jennah list --tenant-id <id> [--status <status>]

Examples

.\jennah list --tenant-id 4a3f92bc-...
.\jennah list --tenant-id 4a3f92bc-... --status RUNNING
Get job details
Shows full details for a single job. Use --history to see state transition history.
.\jennah get <job-id> --tenant-id <id> [--history]

Example

.\jennah get 16f2379d-746b-4e50-986b-f8cb7f1d3ab6 --tenant-id 4a3f92bc-...
Update a job's status
Changes the status of an existing job.
.\jennah update <job-id> --status <status> --tenant-id <id>

Example

.\jennah update 16f2379d-... --status COMPLETED --tenant-id 4a3f92bc-...
Delete a job
Permanently removes a job.
.\jennah delete <job-id> --tenant-id <id>

Example

.\jennah delete 16f2379d-... --tenant-id 4a3f92bc-...

7 Â· Full Workflow Example

Here is a complete example from first setup to deleting a job.

# 1. Set environment variables
$env:JENNAH_PROJECT  = "labs-169405"
$env:JENNAH_INSTANCE = "alphaus-dev"
$env:JENNAH_DATABASE = "main"

# 2. Create a tenant
.\jennah tenant create --name gwenn --email gwenn@example.com
#    â†’ copy the Tenant ID from output, e.g. 4a3f92bc-...

# 3. Submit a job
.\jennah submit --tenant-id 4a3f92bc-...
#    â†’ copy the Job ID from output, e.g. 16f2379d-...

# 4. List all jobs
.\jennah list --tenant-id 4a3f92bc-...

# 5. Get details of a specific job
.\jennah get 16f2379d-... --tenant-id 4a3f92bc-...

# 6. Update its status
.\jennah update 16f2379d-... --status COMPLETED --tenant-id 4a3f92bc-...

# 7. Confirm the update
.\jennah get 16f2379d-... --tenant-id 4a3f92bc-...

# 8. Delete the job
.\jennah delete 16f2379d-... --tenant-id 4a3f92bc-...
